## 1. Webtop 실행 방법

### 사전 준비

- **Docker** 설치
- 터미널(Mac/Linux) 또는 PowerShell(Windows)을 실행.
- 포트 `3000`, `3001`이 다른 프로그램에 사용 중이지 않은지 확인.

```bash
# 포트 사용 여부 확인 (Mac/Linux)
lsof -i :3000
lsof -i :3001
```


### Step 1 · 컨테이너 실행

```bash
docker run -d \
  --name=webtop \
  -e PUID=1000 \
  -e PGID=1000 \
  -e TZ=Etc/UTC \
  -p 3000:3000 \
  -p 3001:3001 \
  --shm-size="1gb" \
  --restart unless-stopped \
  lscr.io/linuxserver/webtop:latest
```

**확인:** 아래 명령어 실행 후 `webtop` 컨테이너가 `Up` 상태인지 확인

```bash
docker ps
```


### Step 2 · 브라우저로 접속

컨테이너가 뜨고 나서 브라우저에서 http://localhost:3000 으로 접속.

**확인:** 브라우저 안에 **리눅스 데스크탑 화면**이 표시되면 정상.


### Step 3 · 기능 동작 확인

- **원격 브라우저 실행**: 바탕화면의 브라우저 아이콘 클릭, 원격 컨테이너 안의 브라우저가 열림
- **인터넷 접속**: 원격 브라우저에서 `https://google.com` 접속, 구글 홈페이지가 로드됨
- **클립보드 동기화**: 로컬에서 텍스트 복사 → 원격 브라우저 주소창에 붙여넣기, 텍스트가 그대로 입력됨
- **한글 입력**: 원격 브라우저 주소창에 한글 입력, 한글이 깨지지 않고 정상 입력됨
- **세션 유지**: 브라우저 탭을 닫고 다시 `localhost:3000` 접속, 이전 상태가 그대로 유지됨


### Step 4 · 종료 및 정리

```bash
# 컨테이너 중지
docker stop webtop

# 컨테이너 삭제
docker rm webtop
```

## 2. 나만의 RBI 시스템 — 기능 요구사항

Webtop을 써보고 나서 저만의 RBI 시스템을 만들 때 구현하고 싶은 기능들입니다. 다 할 수 있을진 모르겠지만 최대한 많이 적어봤습니다.

#### 파일 전송
- 드래그 앤 드롭 방식의 업로드/다운로드 인터페이스

#### 인증 및 세션 관리
- 사용자 인증(Auth) 처리
- 사용자별 독립 컨테이너 세션 할당

#### 컨테이너 오케스트레이션
- **K8s 연동:** 다수 사용자를 위한 컨테이너 자동 스케일링 및 관리

#### 관리자 대시보드 / 정책 관리 페이지
- 사용자별 권한 제어 (예: 파일 전송 비활성화, 특정 URL 접근 차단)
- 정책 템플릿 및 그룹 단위 설정 지원

#### 화면 워터마킹
- 화면 캡처를 통한 정보 유출을 방지하기 위해 사용자 IP나 아이디같은 것을 투명하게 표시

#### 가상 프린팅
- 원격 인쇄 리다이렉션: 원격 컨테이너 내의 인쇄 명령을 가상 프린터 드라이버로 캡처하여 PDF 파일로 변환 후 로컬로 전송
- 출력물 보안 제어: 출력 시 워터마크 강제 삽입 및 인쇄 이력(누가, 언제, 무엇을)에 대한 상세 로그 저장

#### 화면 녹화
- 보안 사고 발생 시 추적을 위해 브라우저를 녹화하거나 모든 클릭/입력 로그를 저장

#### 화상회의 지원 (WebRTC)
- 로컬의 카메라 및 마이크 스트림을 원격 컨테이너로 전달하여 브라우저 내 화상회의 가능하도록 구현


### 남은 계획

앞으로 Webtop 기반 환경을 직접 커스터마이징하며 위 기능들을 하나씩 구현해 나갈 예정입니다. 우선순위는 **사용자 인증 → 프론트엔드/백엔드 → 리다이렉션 기능** 순으로 진행할 계획입니다.
